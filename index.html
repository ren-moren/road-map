Definisi & Cara Kerja
Apa itu Sistem Operasi?
OS adalah program pertama yang berjalan saat komputer dinyalakan (setelah BIOS/UEFI). Tugasnya: mengelola semua hardware dan menyediakan layanan bagi program lain.

Windows Linux macOS Android
Lapisan Abstraksi Lengkap
Aplikasi User (Chrome, VS Code, Python app...)
↕ System Call Interface
Kernel Space — Scheduler · MM · VFS · Networking
↕ Device Driver Interface
Hardware Abstraction Layer (HAL)
↕ Bus (PCIe, USB, SATA...)
Hardware: CPU · RAM · Disk · GPU · NIC
Tanpa OS
Program harus tahu cara bicara langsung ke hardware. Setiap program tulis ulang driver-nya sendiri. Tidak portable, sangat rumit.
Dengan OS
OS menjadi perantara. Program cukup minta lewat system call. OS yang urus hardware. Program jadi portable & aman.

CPU Ring 0 vs Ring 3 — Privilege Levels▼
diagram
Ring 0 (Kernel Mode)
  ├── Akses ke semua instruksi CPU
  ├── Bisa baca/tulis semua alamat memori
  ├── Bisa set interrupt handler
  └── I/O port langsung

Ring 3 (User Mode) — tempat program kamu berjalan
  ├── Tidak bisa akses hardware langsung
  ├── Tidak bisa baca memori proses lain
  ├── HARUS minta lewat system call untuk I/O
  └── Coba akses Ring 0 → SIGSEGV (segfault)

Transisi Ring 3 → Ring 0:
  program memanggil system call (mis: read())
       ↓ CPU trap (software interrupt)
  CPU switch ke Ring 0
       ↓ kernel eksekusi syscall handler
  Kernel return → CPU kembali ke Ring 3
bash — lihat info kernel
uname -r          # versi kernel: 6.5.0-35-generic
uname -a          # info lengkap
lsmod             # modul kernel yang loaded
cat /proc/version # versi + compiler
cat /proc/cpuinfo # info CPU
cat /proc/meminfo # info memori
cat /proc/1/maps  # memory map PID 1

Alur Request I/O Lengkap

↓ kernel handler

Hardware: SSD baca sektor → copy ke RAM via DMA
↓ return ke user space

CFS Scheduler + Context Switch▼
Linux pakai CFS (Completely Fair Scheduler) — red-black tree, O(log n). Context switch ~1-10µs, terjadi ribuan kali per detik di server.
bash
nice -n 10 python3 script.py  # prioritas rendah
renice -n -5 -p 1234          # ubah prioritas proses
ps -eo pid,ni,pri,cmd         # lihat priority
diagram — PCB
PCB (Process Control Block):
  ├── PID, PPID, state
  ├── Program Counter (instruksi berikutnya)
  ├── CPU Registers (rax, rbx, rsp, rbp...)
  ├── Memory Maps (page table pointer)
  ├── Open File Descriptors
  └── Signal handlers

Virtual Memory
Setiap proses punya address space virtual sendiri. MMU translate ke fisik. Proses A & B bisa sama-sama punya "0x1000" tapi menunjuk RAM yang berbeda → isolasi penuh.
Page Table & TLB▼
diagram
Virtual Addr → [MMU + Page Table] → Physical Addr

Page Table:
  Virtual Page 0x1000 → Physical Frame 0xA4F00
  Virtual Page 0x2000 → Physical Frame 0xB2C00

TLB = cache translasi (hit rate ~99%)
Page Fault = akses alamat belum di-map → kernel allocate
Segfault = akses alamat invalid → SIGSEGV

Interprocess Communication
Mengapa IPC?
Setiap proses punya memori terisolasi. IPC = mekanisme resmi OS untuk komunikasi antar proses secara aman.
📄
File
Tulis → file → baca. Simpel, paling lambat (lewat disk).
py
open("tmp.txt","w").write(d)
d = open("tmp.txt").read()
🔗
Pipe
Output → input. Di memori. Terminal pakai |.
bash
ls | grep ".py" | wc -l
🔌
Socket
TCP/UDP. Bisa antar mesin berbeda. Cara browser-server komunikasi.
py
s.connect(("localhost",8000))
s.send(b"GET / HTTP/1.1\r\n")
⚡
Shared Memory
OS map RAM sama ke 2 proses. Tercepat. Perlu mutex.
py
shm = mmap.mmap(fd, 4096)
shm.write(b"data")
📬
Message Queue
Kirim ke antrian. Asinkron — tidak tunggu penerima.
py
q.put("msg"); msg=q.get()
🚦
Mutex
Sinkronisasi — cegah race condition di shared resource.
py
lock.acquire()
  # critical section
lock.release()
Kecepatan IPC
Shared Memory
~ns
Pipe / Msg Queue
~µs
Unix Socket
~µs
File (disk)
~ms
Syscall — C Low Level
Kenapa Penting?
Python, Go, Node.js — semuanya pada akhirnya memanggil syscall yang sama ini. Memahami syscall = memahami apa yang benar-benar terjadi.
File I/O Syscall (C)▼
c
#include <fcntl.h>
#include <unistd.h>
int main() {
    int fd = open("data.txt", O_RDONLY);
    char buf[4096];
    ssize_t n = read(fd, buf, sizeof(buf));
    write(1, buf, n);  // fd=1 adalah stdout
    close(fd);
    return 0;
}
fork() + exec() + wait() (C)▼
c
#include <unistd.h>
#include <sys/wait.h>
int main() {
    pid_t pid = fork();  // duplikasi proses
    if (pid == 0) {
        // CHILD PROCESS
        execl("/bin/ls", "ls", "-la", NULL);
    } else {
        // PARENT PROCESS
        int status;
        wait(&status);  // tunggu child selesai
    }
    return 0;
}

2.2 — FastAPI REST API Lengkap▼
python — main.py
from fastapi import FastAPI, HTTPException
from pydantic import BaseModel
from typing import Optional, List

app = FastAPI(title="My API")

class Item(BaseModel):
    name: str; price: float
    in_stock: bool = True

db: dict = {}; ctr = 1

@app.get("/")
async def root(): return {"status": "ok"}

@app.post("/items", status_code=201)
async def create(item: Item):
    global ctr; db[ctr]=item; ctr+=1
    return {"id": ctr-1, **item.dict()}

@app.get("/items/{id}")
async def get(id: int):
    if id not in db: raise HTTPException(404, "Not found")
    return db[id]

# uvicorn main:app --reload
# http://localhost:8000/docs  ← docs otomatis!
2.3 — Auth JWT + bcrypt▼
python — auth.py
from jose import jwt; from passlib.context import CryptContext
from datetime import datetime, timedelta; import os

SECRET = os.getenv("SECRET_KEY", "ganti-ini!")
pwd = CryptContext(schemes=["bcrypt"], deprecated="auto")

def hash_pw(p): return pwd.hash(p)
def verify_pw(p, h): return pwd.verify(p, h)

def create_token(data: dict):
    exp = datetime.utcnow() + timedelta(minutes=30)
    return jwt.encode({**data, "exp": exp}, SECRET, "HS256")

Docker & Kernel — Namespaces + cgroups▼
Docker BUKAN VM. Container berbagi kernel yang sama dengan host. Isolasi menggunakan fitur kernel Linux.
diagram
namespaces → isolasi VIEW:
  PID ns   → container hanya lihat prosesnya sendiri
  NET ns   → interface network sendiri
  MNT ns   → filesystem tree sendiri
  UTS ns   → hostname sendiri

cgroups → batasi RESOURCE:
  cpu      → max 0.5 core
  memory   → max 512MB
  blkio    → batasi disk I/O

overlay2 → layered filesystem:
  image layers  : read-only (FROM + RUN + COPY)
  container layer: read-write (runtime)
  Copy-on-write : modifikasi tidak ubah layer bawah

Rate Limiting dengan Redis▼
python
from fastapi import Request, HTTPException
from redis import Redis

r = Redis(host='cache', port=6379, decode_responses=True)
LIMIT = 100; WINDOW = 60  # 100 req/menit

async def rate_limit(req: Request):
    key = f"rl:{req.client.host}"
    current = r.get(key)
    if current is None: r.setex(key, WINDOW, 1); return
    if int(current) >= LIMIT:
        raise HTTPException(429, "Too many requests")
    r.incr(key)

Advanced SQL — JOIN, CTE, Window Functions▼
sql
-- LEFT JOIN dengan COUNT
SELECT u.name, COUNT(p.id) AS posts
FROM users u LEFT JOIN posts p ON p.user_id=u.id
GROUP BY u.id;

-- CTE
WITH recent AS (
  SELECT * FROM posts WHERE created_at > NOW()-INTERVAL '7 days'
)
SELECT u.name, COUNT(r.id) FROM users u JOIN recent r ON r.user_id=u.id GROUP BY u.name;

-- Window function: ranking
SELECT name, salary, RANK() OVER (ORDER BY salary DESC) AS rank
FROM employees;

-- Index untuk performance
CREATE INDEX idx_email ON users(email);
EXPLAIN ANALYZE SELECT * FROM users WHERE email='x@y.com';

Redis Cache Pattern▼
python
import redis, json
r = redis.Redis(host='cache', decode_responses=True)

def get_user(user_id: int):
    cached = r.get(f"user:{user_id}")
    if cached: return json.loads(cached)
    user = db_query(user_id)  # query DB
    r.setex(f"user:{user_id}", 3600, json.dumps(user))
    return user

def update_user(user_id, data):
    db_update(user_id, data)
    r.delete(f"user:{user_id}")  # invalidate cache

SQL Injection Prevention▼
python — BAD vs GOOD
# ❌ BAD: SQL Injection vulnerable
query = f"SELECT * FROM users WHERE email='{email}'"

# ✅ GOOD: Parameterized query
query = "SELECT * FROM users WHERE email = %s"
cursor.execute(query, (email,))

# ✅ BETTER: ORM (SQLModel/SQLAlchemy)
session.exec(select(User).where(User.email == email))

CI/CD Pipeline Lengkap

name: CI/CD
on:
  push: { branches: [main] }
  pull_request: { branches: [main] }

jobs:
  test:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v3
      - uses: actions/setup-python@v4
        with: { python-version: '3.12' }
      - run: pip install -r requirements.txt
      - run: pytest --cov=app tests/

  deploy:
    needs: test
    if: github.ref == 'refs/heads/main'
    runs-on: ubuntu-latest
    steps:
      - uses: appleboy/ssh-action@v1
        with:
          host: ${{ secrets.VPS_HOST }}
          username: ${{ secrets.VPS_USER }}
          key: ${{ secrets.SSH_PRIVATE_KEY }}
          script: |
            cd ~/my-api
            git pull origin main
            docker compose up -d --build
            docker system prune -f
            echo "Deploy done: $(date)"
Backup & Recovery
3-2-1 Rule
3 salinan data · 2 media berbeda · 1 off-site. Test restore secara berkala — backup yang tidak pernah di-test = tidak ada backup.
bash — PostgreSQL backup + upload S3
pg_dump -U postgres mydb > backup_$(date +%Y%m%d).sql
psql -U postgres mydb < backup_20260516.sql

# Cron otomatis jam 2 pagi
0 2 * * * pg_dump -U postgres mydb | gzip > /backups/db_$(date +%Y%m%d).sql.gz
# Upload ke S3
aws s3 cp /backups/db_$(date +%Y%m%d).sql.gz s3://my-backups/
