Cara Kerja Tombol Dark / Light Mode
⚡ Cara Kerja Toggle Tema
Tombol di pojok kiri atas mengubah tampilan dari gelap ke terang dan sebaliknya. Berikut cara kerjanya secara teknis:
1. Atribut data-theme di HTML▼
HTML punya atribut data-theme di tag <html>. Nilainya bisa "dark" atau "light". Semua CSS dibaca berdasarkan nilai ini.
html
<!-- default gelap -->
<html data-theme="dark">

<!-- saat diklik tombol, JS mengubahnya jadi -->
<html data-theme="light">
2. CSS Variables per tema▼
Semua warna disimpan di CSS variables. Ada dua blok — :root untuk dark, dan [data-theme="light"] untuk override warna ke versi terang.
css
/* DARK MODE — default */
:root {
  --bg:  #000000;    /* background hitam */
  --bg1: #0a0a0a;   /* card gelap */
  --t1:  #F4F4F5;   /* teks terang */
  --t2:  #A1A1AA;   /* teks abu */
}

/* LIGHT MODE — override semua var */
[data-theme="light"] {
  --bg:  #f8f8f6;   /* background krem */
  --bg1: #ffffff;   /* card putih */
  --t1:  #18181b;   /* teks gelap */
  --t2:  #52525b;   /* teks abu */
}
Semua elemen di halaman pakai var(--bg), var(--t1), dll. Saat data-theme berubah, semua warna otomatis ikut berubah karena pakai variable yang sama.
3. JavaScript Toggle▼
Fungsi JS yang berjalan saat tombol diklik. Mengubah atribut data-theme, update ikon dan label, lalu simpan pilihan ke localStorage agar diingat saat halaman di-refresh.
javascript
function toggleTheme() {
  const html = document.documentElement;
  const isDark = html.getAttribute('data-theme') === 'dark';

  // 1. Ubah atribut data-theme
  html.setAttribute('data-theme', isDark ? 'light' : 'dark');

  // 2. Update ikon dan label tombol
  document.getElementById('themeIcon').textContent = isDark ? '☀️' : '🌙';
  document.getElementById('themeLabel').textContent = isDark ? 'Terang' : 'Gelap';

  // 3. Simpan ke localStorage agar diingat
  localStorage.setItem('theme', isDark ? 'light' : 'dark');
}

// Saat halaman load, ambil tema yang tersimpan
const saved = localStorage.getItem('theme');
if (saved) {
  document.documentElement.setAttribute('data-theme', saved);
  // update ikon sesuai tema tersimpan
}
4. CSS Transition — animasi halus▼
Perubahan warna tidak langsung "pop" — terasa halus karena semua elemen punya transition pada properti warna.
css
body {
  /* animasi 0.3 detik saat background berubah */
  transition: background .3s ease, color .3s ease;
}

.card {
  transition: background .3s, border-color .3s;
}

/* thumb toggle bergerak kanan-kiri */
.toggle-thumb {
  transition: left .3s ease;
  left: 2px;  /* dark: kiri */
}
[data-theme="light"] .toggle-thumb {
  left: 14px;  /* light: kanan */
}
5. localStorage — ingat pilihan user▼
localStorage adalah penyimpanan di browser yang tidak hilang saat halaman ditutup/di-refresh. Berbeda dengan variable JS biasa yang reset setiap load.
javascript
// Simpan
localStorage.setItem('theme', 'light');

// Ambil
localStorage.getItem('theme'); // → "light"

// Hapus
localStorage.removeItem('theme');

// Bisa cek di browser:
// DevTools → Application → Local Storage
💡 Ringkasan alur: Klik tombol → JS ubah data-theme di HTML → CSS selector [data-theme="light"] aktif → semua var(--bg), var(--t1) dll berubah nilai → semua elemen otomatis update warna dengan animasi .3s.

from fastapi import FastAPI, HTTPException
from pydantic import BaseModel

app = FastAPI()

class Item(BaseModel):
    name: str; price: float

db: dict = {}; ctr = 1

@app.post("/items", status_code=201)
async def create(item: Item):
    global ctr; db[ctr]=item; ctr+=1
    return {"id": ctr-1, **item.dict()}

Rate Limiting + Auth▼
python
from fastapi import Depends, HTTPException
from fastapi.security import HTTPBearer
from redis import Redis

r = Redis(host='cache', decode_responses=True)
sec = HTTPBearer()

async def rate_limit(req):
    key = f"rl:{req.client.host}"
    n = r.get(key)
    if not n: r.setex(key, 60, 1); return
    if int(n) >= 100:
        raise HTTPException(429, "Too many requests")
    r.incr(key)
